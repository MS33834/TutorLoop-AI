## 1. 阶段规划

### Phase 1：AI 网关与流式基础闭环（2-3 周）

**目标**：跑通「学生提问 → AI 回答」的最小闭环。

- [x] FastAPI 项目脚手架、日志、统一异常处理。
- [x] 多 Key 健康池、加权轮询、熔断降级机制。
- [x] SSE 流式对话接口 `/api/rooms/:slug/ask`。
- [x] Vue3 PWA 基础框架 + 聊天界面 + SSE 渲染。
- [x] 本地模型（vLLM / Ollama）兜底验证。

**验收标准**：
- 多 Key 故障自动切换 < 5s。
- 本地模型兜底成功率 100%。
- 首字延迟 < 1s（测试环境）。

---

### Phase 2：多模态交互与知识提取（3-4 周）

**目标**：实现视频时间轴提问与知识图谱自动构建。

- [x] 自定义视频播放器组件，支持截图 + 时间戳捕获。
- [x] 上传视频 / 图片 / PDF，触发知识图谱构建 Agent。
- [x] Neo4j 接入与图谱可视化编辑。
- [x] 多模态 RAG：视频帧向量 + 文本片段召回。
- [x] 老师端房间与课程管理后台。

**验收标准**：
- 截图提问端到端延迟 < 3s。
- 知识图谱构建可用，节点准确率经人工校验 > 80%。
- RAG 召回 Top-5 准确率 > 70%。

---

### Phase 3：追踪算法与自适应闭环（3-4 周）

**目标**：跑通「看视频 → 提问 → 更新掌握度 → 推荐下一步」完整闭环。

- [x] BKT 掌握度引擎实现。
- [x] 学习行为埋点与上报（观看时长、回看、求助、回答正确性）。
- [x] 掌握度雷达图前端实时展示。
- [x] 自适应路径推荐算法。
- [x] 学习报告生成。

**验收标准**：
- 掌握度更新延迟 < 500ms。
- 推荐节点符合前置依赖约束。
- 学生完成一次完整学习闭环无阻塞 Bug。

---

### Phase 4：部署上线与效果评估（2-3 周）

**目标**：生产可用，沉淀简历亮点。

- [x] 前端 Vercel / Cloudflare Pages / Docker 部署配置。
- [x] 后端 Docker + Sealos / Render 部署配置。
- [x] 监控告警（Sentry + Prometheus 基础指标）。
- [ ] 小规模真实教学场景测试（10-20 名学生）。
- [ ] 性能优化与成本压测。

**验收标准**：
- 线上可用性 > 99%。
- 移动端核心流程无阻塞。
- 收集到至少 5 份可用性反馈。

## 3. 风险与应对

| 风险 | 影响 | 应对策略 |
| --- | --- | --- |
| 多模态 API 成本高 | 预算超支 | 模型分层路由、本地模型兜底、截图压缩、缓存热点答案 |
| 首字延迟不达标 | 体验差 | SSE 预连接、热点视频预索引、多 Key 健康池、本地兜底 |
| 知识图谱构建质量差 | 推荐失效 | 人工校验流程、多模型投票、置信度阈值过滤 |
| 移动端兼容性 | 用户流失 | 真机测试、渐进增强、微信内置浏览器专项适配 |
| 数据隐私合规 | 合规风险 | 数据脱敏、截图过期清理、最小权限原则、隐私协议 |

## 5. 简历亮点包装

1. **全栈云原生架构**：设计并落地 Vue3(PWA) + FastAPI + Neo4j + Supabase 的多模态教育 Agent，前端 CDN 全球加速，学生零安装即开即用。
2. **高可用 AI 网关**：自研多 Key 健康池 + 加权轮询 + 熔断降级机制，支持云端模型与本地 vLLM/Ollama 自动兜底，SSE 流式首字延迟 < 1s。
3. **多模态学习交互**：实现视频时间戳自动捕获与画面截图融合，结合多模态 RAG 提供苏格拉底式引导回答，而非直接抛出答案。
4. **教育算法工程化**：自研贝叶斯知识追踪（BKT）引擎，融合 Neo4j 知识图谱与自适应路径规划，跑通「看视频-提问-更新掌握度-推荐下一步」完整闭环。
