## 1. 项目定位

TutorLoop-AI 是一个**开箱即用的多模态自适应学习 AI Agent 系统**，面向 K12 与职业教育场景。核心目标：

- **老师端**：上传视频、图片、文本等学习资料，配置知识点范围与评分标准，系统自动构建知识图谱并生成可分享的学习房间。
- **学生端**：无需安装 App，点击链接即可进入学习；在观看视频过程中可随时基于当前画面提问；AI 以苏格拉底式引导回答问题，并实时追踪掌握度、动态调整学习路径。

---

## 2. 核心价值主张

1. **零安装、零配置**：学生端基于 PWA，微信或浏览器直接打开，体验接近原生 App。
2. **多模态上下文感知**：自动捕获视频时间戳与画面截图，融合文本问题进行理解。
3. **知识体系闭环**：从资料抽取知识图谱 → 实时追踪掌握度 → 自适应推荐最短补弱路径。
4. **高并发高可用 AI 网关**：多 API Key 负载均衡、健康检查、熔断降级与本地模型兜底，SSE 流式响应保障体验。

---

## 3. 目标用户

| 角色 | 核心诉求 |
| --- | --- |
| 教师 / 教育机构 | 快速生成个性化辅导课程，追踪班级与个体掌握度 |
| 学生 | 利用碎片化时间自主学习，获得针对性引导而非直接答案 |
| 家长 | 查看学习报告，直观了解薄弱知识点与学习投入 |

## 4. 团队角色与职责

| 角色 | 核心职责 | 关键产出 |
| --- | --- | --- |
| **产品负责人 / 学习科学顾问** | 定义知识点拆分粒度、掌握度阈值、评分标准；验证教育闭环有效性 | PRD、知识图谱标注规范、评估指标体系 |
| **Tech Lead（全栈架构）** | 把控 Vue3 + FastAPI + LangGraph 整体架构；制定接口与数据规范 | 架构图、API 规范、技术选型决策 |
| **AI 算法工程师** | 多模态 RAG、知识抽取 Agent、BKT 算法实现 | Agent 工作流、BKT 引擎、评测脚本 |
| **后端工程师** | FastAPI 网关、SSE 流式转发、数据库 / 图库、部署 | 网关代码、数据库 Schema、Dockerfile |
| **前端工程师** | PWA 视频播放器、截图提问、雷达图、SSE 渲染 | 前端组件库、离线缓存策略 |
| **数据 / 运维工程师** | Supabase / Neo4j 运维、日志监控、成本控制 | 部署脚本、监控看板、成本报表 |
| **QA / 测试** | 端到端测试、教育效果回归测试、多端兼容性 | 测试用例、自动化脚本 |
