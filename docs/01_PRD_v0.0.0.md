## 1. 产品形态

TutorLoop-AI 为 **Web PWA 应用**，采用移动端优先设计。老师端与学生端为同一应用的不同权限视图：

- **学生端**：免安装，点击房间链接即可学习。
- **老师端**：登录后进入课程与房间管理后台。

---

## 2. 用户旅程

### 2.1 老师旅程

1. 登录后台。
2. 创建课程，上传视频、图片、PDF 等资料。
3. 配置知识点范围、掌握度阈值（默认 0.8）与评分标准。
4. 系统自动构建知识图谱，老师可人工修正节点与依赖关系。
5. 生成学习房间 URL 与二维码，分享给学生。
6. 查看班级与个体学习报告，掌握整体学情。

### 2.2 学生旅程

1. 点击链接或扫码进入学习房间。
2. 登录或匿名进入。
3. 观看视频 / 阅读资料。
4. 遇到疑问点击「截图提问」，系统自动捕获当前帧与时间戳。
5. 输入问题，AI 以苏格拉底式引导回答。
6. 侧边栏掌握度雷达图实时更新。
7. 系统根据掌握度推荐下一步学习片段。
8. 完成学习后查看个人学习报告。

## 3. 功能模块

### 3.1 学习房间管理

- 创建 / 编辑 / 删除学习房间。
- 上传课程视频、参考图片、PDF 文档。
- 配置知识点列表、掌握度阈值、评分标准。
- 生成分享链接与二维码。
- 设置房间有效期与访问密码（可选）。

### 3.2 多模态视频学习

- 自定义视频播放器，支持播放、暂停、进度拖动、倍速播放。
- 时间轴进度与后端同步，记录观看行为。
- 点击「截图提问」自动捕获当前帧并记录时间戳。
- 展示字幕 / ASR 文本，支持关键词高亮。

### 3.3 AI 苏格拉底式答疑

- 接收输入：视频帧截图、当前时间戳、学生文本问题。
- 多模态 RAG 检索：视频帧向量、图谱节点、参考资料。
- 默认以引导式、反问式回答，帮助学生自主推导。
- 提供「我要看答案」兜底按钮，避免学生卡死。
- 答案通过 SSE 流式输出，呈现打字机效果。

### 3.4 掌握度雷达图

- 实时展示各知识点掌握概率 `P(L)`。
- 颜色区分：已掌握（≥ 阈值）、学习中、未掌握。
- 点击知识点节点可查看定义、前置依赖与历史交互。

### 3.5 自适应路径推荐

- 推荐逻辑：只推荐「前置知识点已掌握但自身未掌握」的节点。
- 展示推荐理由，例如：「该知识点是『二元一次方程』的前置，你当前掌握度 45%。」
- 推荐结果一键跳转至对应视频片段。

### 3.6 学习报告

- 观看热力图（时间轴上的停留与回看分布）。
- 提问次数与问题类型分布。
- 各知识点掌握度变化曲线。
- 薄弱知识点清单与针对性练习建议。

## 4. 非功能需求

### 4.1 性能

- 首屏加载 < 2s（3G 网络）。
- 视频首帧加载 < 1s。
- AI 首字延迟 < 1s。
- 雷达图更新延迟 < 500ms。

### 4.2 可用性

- PWA 离线缓存核心静态资源。
- 弱网环境下可提交问题，网络恢复后自动重试。
- 错误状态有明确提示与重试入口。

### 4.3 安全

- 房间 URL 支持密码保护与有效期限制。
- 学生数据脱敏存储，截图设置过期自动清理。
- 全站 HTTPS，敏感接口 JWT 鉴权。

### 4.4 移动端

- 适配 iOS Safari、微信内置浏览器、Android Chrome。
- 支持添加到桌面、横竖屏切换、手势操作。
- 截图提问适配移动端截图/拍照上传。

## 5. 术语表

| 术语 | 说明 |
| --- | --- |
| 学习房间 | 老师创建的独立学习空间，通过 URL 分享 |
| 知识点 | 课程中需要掌握的最小能力单元 |
| 掌握度 | 学生掌握某知识点的概率，通常用 BKT 计算 |
| BKT | Bayesian Knowledge Tracing，贝叶斯知识追踪 |
| DKT | Deep Knowledge Tracing，深度知识追踪 |
| SSE | Server-Sent Events，服务端推送技术 |
| PWA | Progressive Web App，渐进式 Web 应用 |
| RAG | Retrieval-Augmented Generation，检索增强生成 |
| Agent | 具备特定能力的 AI 模块，由 LangGraph 编排 |
