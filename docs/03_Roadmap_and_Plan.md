## 1. 已完成阶段总结（Phase 1-3 + Phase 4 部分）

### Phase 1：AI 网关与流式基础闭环 ✅

**核心成果**：
- FastAPI 项目脚手架，统一异常处理、请求 ID 中间件、安全头、CSP
- 多 Key 健康池：加权轮询（配额 + RTT + 成功率）、熔断降级（3 次错误转 offline）、本地模型兜底
- SSE 流式对话接口 `/api/rooms/:slug/ask`，支持打字机效果
- Vue3 PWA 基础框架 + 聊天界面 + SSE 渲染
- 网关健康探测在 web 进程启动（不依赖 worker）

**简历亮点**：自研多 Key 健康池 + 加权轮询 + 熔断降级机制，支持云端模型与本地 vLLM/Ollama 自动兜底

### Phase 2：多模态交互与知识提取 ✅

**核心成果**：
- 自定义视频播放器：播放/暂停/进度拖动/倍速/全屏/音量/字幕高亮/截图提问
- 上传视频/图片/PDF，触发知识图谱构建 Agent（异步 ARQ 任务入队）
- Neo4j 接入与图谱可视化编辑（Cytoscape），关系类型白名单映射
- 多模态 RAG：视频帧向量检索 + 截图 VLM 描述 + 文本片段召回 + pgvector 混合检索
- 老师端房间与课程管理后台，房间密码/有效期/二维码分享
- SRT/VTT 字幕解析（修复无序号 SRT 丢首行 bug）

**简历亮点**：实现视频时间戳自动捕获与画面截图融合，结合多模态 RAG 提供苏格拉底式引导回答

### Phase 3：追踪算法与自适应闭环 ✅

**核心成果**：
- BKT 掌握度引擎：标准贝叶斯知识追踪公式，每节点 P(T)/P(G)/P(S) 参数化，除零保护
- 掌握度历史快照表（MasterySnapshot），支持时序曲线查询
- 学习行为埋点：观看时长、回看、求助、回答正确性、视频进度同步
- 掌握度雷达图：Top8 紧迫度筛选 + 显示全部切换 + 三色掌握度区分
- 自适应路径推荐：前置依赖约束 + 迭代式拓扑排序深度计算 + 推荐理由 + 视频跳转
- 学习报告：个人报告（热力图 + 掌握度曲线 + 提问类型分布 + 薄弱知识点）+ 班级报告
- 苏格拉底引导：三档掌握度提示词 + LLM 评估 + 短答案兼容 + 匿名用户动态 band

**简历亮点**：自研贝叶斯知识追踪（BKT）引擎，融合 Neo4j 知识图谱与自适应路径规划，跑通完整学习闭环

### Phase 4（已完成部分）✅

**核心成果**：
- 前端 Docker + Vercel + Cloudflare Pages 部署配置
- 后端 Docker + Sealos + Render + Fly.io 部署配置
- 监控告警：Sentry + Prometheus 基础指标
- PWA 完整配置：manifest 图标（192/512/maskable）、iOS meta、Workbox runtimeCaching
- CI 流水线：后端 lint + 格式 + 类型 + 覆盖率 + 迁移校验；前端 test + build + 覆盖率
- 安全加固：refresh cookie 动态 Secure、CSP style-src unsafe-inline、session token TTL
- 全面 bug 修复：SRT 解析、class_report SQL、entry_count 递减、过期房间清理、并发竞态等 50+ 项

## 2. 新开发计划（Phase 4 续 ~ Phase 7）

### Phase 4 续：真实场景验证与性能优化

**目标**：生产可用，通过真实教学场景验证。

- [ ] 4.1 配置真实 LLM/VLM API Key（DeepSeek / Qwen），在 Docker 环境跑通全链路
- [ ] 4.2 编写端到端（E2E）自动化测试脚本，覆盖核心学习闭环
  - [ ] 4.2.1 老师创建课程 → 上传视频 → 构建知识图谱 → 创建房间
  - [ ] 4.2.2 学生进入房间 → 观看视频 → 截图提问 → AI 回答 → 掌握度更新
  - [ ] 4.2.3 查看学习报告 → 推荐下一步 → 跳转学习
- [ ] 4.3 性能压测与优化
  - [ ] 4.3.1 SSE 首字延迟压测（目标 < 1s）
  - [ ] 4.3.2 并发用户压测（目标 50 并发不降级）
  - [ ] 4.3.3 数据库慢查询排查与索引优化
  - [ ] 4.3.4 前端首屏加载优化（代码分割、懒加载、预连接）
- [ ] 4.4 小规模真实教学场景测试（10-20 名学生）
  - [ ] 4.4.1 准备测试课程内容（至少 3 个学科）
  - [ ] 4.4.2 招募测试学生，收集可用性反馈
  - [ ] 4.4.3 根据反馈迭代优化 UI/UX
- [ ] 4.5 移动端真机兼容性测试
  - [ ] 4.5.1 iOS Safari 测试（PWA 安装、视频播放、截图提问）
  - [ ] 4.5.2 Android Chrome 测试
  - [ ] 4.5.3 微信内置浏览器专项适配
- [ ] 4.6 成本压测与优化
  - [ ] 4.6.1 统计单用户平均 API 调用次数与成本
  - [ ] 4.6.2 实现热点答案缓存（Redis），降低重复提问成本
  - [ ] 4.6.3 模型分层路由优化（简单问题用小模型）

**验收标准**：
- 线上可用性 > 99%
- 移动端核心流程无阻塞
- SSE 首字延迟 < 1s
- 收集到至少 5 份可用性反馈
